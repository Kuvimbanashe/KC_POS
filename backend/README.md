# POS Django Backend (Modular + Deployment Ready)

## App structure
Each domain has its own Django app:
- `businesses` ‚Üí tenant business profiles
- `users` ‚Üí staff users
- `products` ‚Üí catalog + barcode lookup
- `sales` ‚Üí sales + sale items + stock decrement logic
- `purchases` ‚Üí purchases + stock increment logic
- `expenses` ‚Üí expenses
- `assets_mgmt` ‚Üí fixed assets
- `reports` ‚Üí dashboard, sales, inventory, expenses, profit/loss endpoints

## REST endpoints
Core:
- `/api/businesses/`
- `/api/auth/register-business/`
- `/api/auth/login/`
- `/api/users/`
- `/api/products/` (SKU generated by backend if omitted)
- `/api/products/lookup-by-barcode/?barcode=<code>`
- `/api/sales/`
- `/api/purchases/`
- `/api/expenses/`
- `/api/assets/`

Reports:
- `/api/reports/dashboard/`
- `/api/reports/sales/?start=YYYY-MM-DD&end=YYYY-MM-DD`
- `/api/reports/inventory/`
- `/api/reports/expenses/?start=YYYY-MM-DD&end=YYYY-MM-DD`
- `/api/reports/profit-loss/?start=YYYY-MM-DD&end=YYYY-MM-DD`

Health:
- `/healthz/`

## Local setup
```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

## Deployment readiness
The backend is configured for production-oriented deployment:
- `gunicorn` Procfile for process managers/platforms.
- `whitenoise` static serving.
- `dj-database-url` for `DATABASE_URL` parsing (supports Postgres).
- security toggles based on `DJANGO_DEBUG`.
- `.env.example` with required env variables.

Example env:
```bash
DJANGO_SECRET_KEY=replace-with-secure-value
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=your-domain.com,localhost
DJANGO_CORS_ALLOW_ALL=False
DJANGO_TIME_ZONE=UTC
DATABASE_URL=postgres://user:pass@host:5432/dbname
```

## Expo integration
Set app environment:

```bash
EXPO_PUBLIC_API_BASE_URL=http://<your-machine-ip>:8000/api
EXPO_PUBLIC_ENABLE_BACKEND_SYNC=true
# users are scoped by businessId from auth login
```

Then restart Expo.
